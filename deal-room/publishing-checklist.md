# Publishing Checklist — 6 Channels

## Purpose

This checklist converts the sales pack into live marketplace listings and outbound-ready profiles.

Channels:

1. Upwork
2. Contra
3. LinkedIn
4. TenChat
5. Kwork
6. FL.ru

## Shared assets to prepare

Before publishing, prepare:

- profile photo;
- short bio;
- portfolio link: deal-room landing page;
- sample audit report;
- GitHub portfolio screenshots;
- simple before/after graphic;
- service pricing;
- payment and communication preference;
- response templates.

## 1. Upwork

Use file: `publish-upwork.md`

Actions:

- create or update profile headline;
- create Project Catalog offer;
- add three packages;
- add buyer requirements;
- add gallery images;
- submit for review.

Priority offer:

AI Repo Rescue Diagnostic.

## 2. Contra

Use file: `publish-contra.md`

Actions:

- update profile positioning;
- add service: AI Repo Rescue Sprint;
- add service: Telegram Automation & HR Autopilot;
- add portfolio items;
- add landing page link.

## 3. LinkedIn

Use file: `publish-linkedin.md`

Actions:

- update headline;
- update About section;
- create Services Page;
- add featured post about Repo Rescue;
- add featured post about HR Autopilot;
- start direct founder outreach.

## 4. TenChat

Use file: `publish-tenchat.md`

Actions:

- update profile positioning;
- add services;
- publish HR Autopilot post;
- publish Repo Rescue post;
- reply to comments with intake link.

## 5. Kwork

Use file: `publish-kwork.md`

Actions:

- create service 1: repo diagnostic;
- create service 2: Telegram hiring funnel;
- set buyer requirements;
- add clear fixed-scope deliverables;
- keep upsells compliant with platform rules.

## 6. FL.ru

Use file: `publish-flru.md`

Actions:

- update profile headline;
- add profile description;
- add services;
- respond to relevant projects;
- use first-response template.

## Daily publishing rhythm

Day 1:

- Upwork
- Contra
- LinkedIn

Day 2:

- TenChat
- Kwork
- FL.ru

Day 3:

- 30 direct messages to founders
- 30 direct messages to HR / operations leads
- improve listing copy based on replies

## Response rule

Every reply should push to one of two entry products:

1. Repo Diagnostic
2. HR Autopilot Pilot

Do not sell vague development hours.

## Tracking table columns

- Platform
- Listing URL
- Date published
- Offer
- Price
- Status
- Leads received
- Calls booked
- Paid diagnostics
- Paid pilots
- Notes
