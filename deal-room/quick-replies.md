# Quick Replies for Incoming Leads

## 1. Repo Rescue — interested lead

Thanks. The cleanest first step is a Repo Diagnostic.

I review the current repository state, identify launch blockers, prepare a risk map and give a rescue sprint recommendation.

Please send:

1. repository link or archive;
2. short product description;
3. what currently does not work;
4. target demo/launch date;
5. current stack if known.

After that I will confirm whether the diagnostic scope is a fit.

## 2. Repo Rescue — asks for free review

I can do a short fit check for free, but a real technical review requires a paid diagnostic because it includes structured analysis, blocker mapping and written recommendations.

The diagnostic starts at $500–$1,500 depending on scope and urgency.

## 3. Repo Rescue — asks for guarantee

I do not promise unlimited fixes before seeing the repository. The diagnostic is designed to define the real state first.

What I can commit to: a clear report, blocker map, risk assessment and practical rescue recommendation. Fixes are scoped after that.

## 4. Repo Rescue — wants full build

Possible, but I would not start with a full build immediately. The right first step is a diagnostic, then a fixed rescue or build scope based on evidence.

This avoids underestimating hidden blockers.

## 5. HR Autopilot — interested lead

Great. The best starting format is a 14-day pilot for one vacancy and one city.

Please send:

1. vacancy;
2. city;
3. current candidate sources;
4. approximate responses per day;
5. who handles candidates now;
6. where statuses are tracked;
7. target metric for the pilot.

After that I will map the funnel and confirm scope.

## 6. HR Autopilot — asks if it replaces recruiters

No. The system does not replace recruiters. It gives them a cleaner queue, candidate statuses, structured screening and fewer manual follow-ups.

The goal is to reduce losses between response and shift start.

## 7. HR Autopilot — asks for free test

I can show the pilot structure and ask diagnostic questions for free. A real configured pilot is paid because it includes funnel setup, candidate questions, status logic and reporting.

The starting pilot is 150,000 RUB for one vacancy and one city.

## 8. Price anchoring

The entry product is intentionally bounded: diagnostic or pilot first, larger sprint only after we see the actual scope.

This protects both sides from vague development work.

## 9. Call booking reply

Good fit. The next step is a 25–30 minute discovery call.

Agenda:

1. current state;
2. target result;
3. blockers;
4. scope;
5. price and timeline;
6. next step.

## 10. Proposal sent follow-up

Quick follow-up. The proposed format keeps the scope bounded and focused on a measurable output. If the timing works, the next step is approval of the diagnostic/pilot and kickoff inputs.
